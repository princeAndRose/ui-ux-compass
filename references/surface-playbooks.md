# Surface Playbooks

Use this as the first reference when the user asks for a page, screen, flow, or product surface. Pick the closest surface, then adapt to repo conventions.

| Surface | Role | Core tasks | Default archetype | P0 / P1 / P2 | Density | Required states | Common anti-patterns | Review focus | Implementation constraints |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| Dashboard | Monitor status and spot action needs | Scan KPIs, compare segments, open details | Summary plus drilldown | P0 health/status; P1 trends and drivers; P2 logs | Medium-high | loading, empty, error, stale data | equal card grid, vanity charts, hidden filters | scan path, chart legibility, actionability | use real units, stable table/chart sizing |
| Workspace | Repeated daily work surface | create, triage, switch context, resume | app shell with split work area | P0 active work; P1 navigation/context; P2 secondary tools | High | loading, empty, error, disabled, unsaved | decorative hero, low-density cards, unclear active item | task speed, keyboard path, state persistence | predictable nav, no layout shift |
| Editor | Produce or manipulate content | edit, preview, configure, publish | split pane or canvas with inspector | P0 work canvas; P1 inspector/preview; P2 history/help | Medium-high | saving, saved, error, dirty, disabled | controls buried below fold, no save status | affordances, undo safety, focus modes | preserve content area and shortcuts |
| Settings | Configure account/product behavior | review current value, change, confirm | grouped form sections | P0 current setting and primary action; P1 explanations; P2 audit/help | Medium | loading, error, success, disabled | flat long form, destructive actions near routine actions | grouping, defaults, confirmation | label clearly, separate dangerous actions |
| Admin | Operate system data and permissions | search, filter, act, audit | data table plus detail drawer | P0 table/search/action; P1 detail; P2 audit metadata | High | loading, empty, error, permission denied | unclear bulk action scope, no audit visibility | safety, filtering, row density | expose ids/status, confirm destructive actions |
| List-detail | Browse many items and act on one | filter, select, compare, inspect | master-detail split | P0 list and selected detail; P1 filters; P2 metadata | Medium-high | empty, filtered-empty, loading, error, selected | detail route loses selection, list jumps | selection clarity, responsive collapse | stable item identity, URL/deep-link if possible |
| Form or wizard | Collect structured input | enter, validate, review, submit | single form or stepper | P0 current required fields; P1 progress/help; P2 examples | Medium | validation, loading, error, success, disabled | too many fields per step, no recovery path | validation timing, progress clarity | preserve input, summarize before submit |
| Onboarding | Help first-time users reach value | choose path, connect data, create first item | guided checklist or focused wizard | P0 next best action; P1 progress; P2 education | Low-medium | skipped, blocked, loading, error | tour without task value, too much copy | time-to-value, skip/resume | make defaults safe, avoid modal prison |
| Landing | Explain offer and drive conversion | understand value, trust proof, convert | hero plus proof sections | P0 offer and CTA; P1 proof; P2 details | Low-medium | responsive, form error, loading | generic SaaS hero, vague headline, no proof | message clarity, credibility, mobile first viewport | real product image or concrete visual proof |
| Pricing | Convert plan comparison | compare, select, understand limits | comparison grid | P0 plan fit and CTA; P1 limits; P2 FAQ | Medium | loading, billing error, selected plan | hidden fees, equal visual weight for all plans | plan hierarchy, trust, edge cases | accessible tables, clear terms |
| Empty/loading/error | Preserve trust when data is absent | explain, recover, retry, create | state panel in the real layout slot | P0 status and next action; P1 reason; P2 support detail | Low | loading, empty, filtered-empty, error | cute empty art without action, skeleton mismatch | action clarity, copy specificity | do not resize container across states |
| Data table/filter | Explore and act on records | search, filter, sort, bulk action | dense table with filter bar | P0 rows and primary filters; P1 columns/actions; P2 saved views | High | loading, empty, filtered-empty, error, selected | filters hidden, tiny hit targets, no row action hierarchy | density, scan columns, filter feedback | sticky headers when useful, preserve filters |
| Modal/drawer/command palette | Interrupt or accelerate a focused task | confirm, edit details, invoke command | overlay, side drawer, or command menu | P0 active decision/input; P1 context; P2 escape/help | Low-medium | loading, error, disabled, keyboard focus | modal for complex workflows, no escape path | focus trap, keyboard use, dismissal safety | keep overlay purpose narrow |
