# Layout Patterns

Pick layout patterns from user task and information hierarchy, not from decoration preference.

## Workspace

Best for frequent operation.

Structure:

- Persistent navigation or context rail.
- P0 work area.
- P1 list, queue, or controls.
- P2 details or helper panel.

Risks:

- Overloading the first viewport.
- Making every panel visually equal.

## Dashboard

Best for scanning status and deciding what to do next.

Structure:

- P0 summary row or hero metric.
- P1 trend, table, or alert list.
- P2 secondary diagnostics.
- Clear next action.

Risks:

- Decorative metrics without decisions.
- Card soup.

## Editor

Best for creating or modifying structured content.

Structure:

- Primary canvas or form.
- Contextual controls.
- Preview or validation.
- Save/publish action with state feedback.

Risks:

- Controls competing with content.
- Missing dirty, error, disabled, and success states.

## Settings

Best for occasional configuration.

Structure:

- Clear grouping.
- Labels and helper text.
- Safe destructive actions.
- Feedback after saving.

Risks:

- Hiding dangerous effects.
- Overusing modals.

## Landing Page

Best for persuasion.

Structure:

- First viewport communicates product or offer.
- Primary proof appears early.
- Clear CTA.
- Real product, place, or object signal.

Risks:

- Generic hero.
- Atmospheric imagery with no inspectable product.
- Too many vague value propositions.
